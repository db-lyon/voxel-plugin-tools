# ue-mcp-plugin-voxel-plugin — Real API TODO

Generated by reading every public header under `../Vale/Plugins/Voxel/Source/`.
Citations are `Module/Public/SubDir/File.h:line` relative to the Voxel plugin root.

> **State of the current ship.** `src/tasks/{ScatterMeshes,SpawnStamps,BakeHeightmap}.ts`
> are slop. They call ue-mcp tasks with wrong parameter names (`graphPath` should be
> `assetPath`, `sourceNodeId` should be `sourceNode`, no `nodeId` is ever returned),
> they pass fabricated PCG node-type strings (`VoxelSampler`, `VoxelStamp` —
> these classes do not exist; the real ones are below), and `BakeHeightmap`
> calls a Python function `unreal.VoxelBlueprintLibrary.get_voxel_height_at(x,y)`
> that does not exist. Delete the lot once the replacements below land.

## Ground-truth API surface (what actually exists)

### A. Voxel World — runtime lifecycle
`Voxel/Public/VoxelWorld.h`. `AVoxelWorld` is the actor users drop in the level.
All UFUNCTIONs are non-static, called on a found actor.

| Function | Returns | Notes |
|---|---|---|
| `IsRuntimeCreated()` | bool | Has the streaming runtime been built |
| `IsVoxelWorldReady()` | bool | A complete state has been computed at least once |
| `IsProcessingNewState()` | bool | A regen is in flight |
| `GetProgress()` | float | 0..1 progress of the in-flight state, else 1 |
| `GetNumPendingTasks()` | int32 | Queue depth |
| `CreateRuntime()` | void | Mandatory if `bCreateRuntimeOnBeginPlay` is false |
| `DestroyRuntime()` | void | |
| `OnInvalidated(FOnVoxelWorldEvent)` | void | Fires when a new state starts computing |
| `OnNextStateRendered(FOnVoxelWorldEvent)` | void | Fires once when pending state finishes |
| `BindOnRuntimeCreated(...)` / `BindOnRuntimeDestroyed(...)` | void | Lifecycle events |

Editable properties of interest: `VoxelSize`, `LayerStack` (TObjectPtr<UVoxelLayerStack>),
`MegaMaterial`, `bEnableNanite`, `bEnableLumen`, `bEnableRaytracing`,
`bCreateRuntimeOnBeginPlay`, `bWaitOnBeginPlay`, `bWaitForPCG`, collision
(`VisibilityCollision`, `InvokerCollision`), navigation toggles, Nanite tess.

### B. Voxel data queries — read heights/metadata from the runtime
`Voxel/Public/VoxelQueryBlueprintLibrary.h` (private surface) and the auto-generated
synchronous K2 wrappers at `Voxel/Public/VoxelQueryBlueprintLibrary_K2.h` —
`UVoxelQueryBlueprintLibrary_BlueprintOnly`. **This is the path the broken
`BakeHeightmap` should have used.**

| K2 function | Signature | Use |
|---|---|---|
| `QueryVoxelLayer` | `(bSuccess, Result, World, FVoxelStackLayer, FVector Position, [metadatas], GradientStep=100)` | Sync sample of one point. Returns `FVoxelQueryResult { float Value, FVector Normal, surface types, metadata→value map }` |
| `MultiQueryVoxelLayer` | `(bSuccess, Results[], World, layer, positions[], [metadatas], GradientStep=100)` | Sync batch — N positions in one call |
| `QueryVoxelLayerAsync` / `MultiQueryVoxelLayerAsync` | Latent variants with `FLatentActionInfo` | Same, but doesn't block |
| `ExportVoxelDataToRenderTarget` | `(bSuccess, World, RenderTarget, FVoxelStackHeightLayer Layer, FVoxelBox2D ZoneToQuery, FVoxelColorQuery Query)` | Render heights or metadata into an RT2D over a 2D world-space box. The right primitive for heightmap baking. |
| `ExportVoxelDataToRenderTargetAsync` | Latent | |
| `MakeConstant(float)` / `QueryHeight()` / `QueryMetadata(meta, channel)` / `QueryColorMetadata(meta)` | Pure | Build `FVoxelFloatQuery` / `FVoxelColorQuery` payloads for the Export call |

### C. Sculpt — runtime terrain modification
`Voxel/Public/Sculpt/Volume/VoxelVolumeSculptBlueprintLibrary.h`
`Voxel/Public/Sculpt/Height/VoxelHeightSculptBlueprintLibrary.h`
Both take their corresponding sculpt actor (`AVoxelVolumeSculptActor` or
`AVoxelHeightSculptActor`) as the target. Both return `FVoxelFuture` (async).

Volume sculpt (3D, `FVector Center`):
`PaintSurface`, `SculptAngle`, `Flatten`, `Smooth`, `SculptSphere`, `SculptCube`,
`SculptSurface`, `ApplySculptGraph`, `ClearSculptData`, `ClearSculptCache`,
`K2_GetSave(FVoxelVolumeSculptSave&, …)`, `LoadFromSave(…)`,
plus `IsValidSave` / `IsCompressedSave` / `GetSaveSize` pures.

Height sculpt (2D, `FVector2D Center`):
`PaintSurface`, `Flatten`, `Smooth`, `SculptHeight`, `ApplySculptGraph`,
`ClearSculptData`, `ClearSculptCache`, `K2_GetSave`, `LoadFromSave`, same pures.

Modes: `EVoxelSculptMode { Add, Remove }`, `EVoxelLevelToolType { Additive, Subtractive, Both }`,
`EVoxelSDFMergeMode { Override, Union, Intersection, … }`.

### D. Stamps & stamp components — placed at runtime
- `AVoxelStampActor` (`Voxel/Public/VoxelStampActor.h`) — `GetStamp`, `SetStamp`, `UpdateStamp`.
- `UVoxelStampComponent` (`VoxelStampComponent.h`) — `GetStamp`, `SetStamp`, `UpdateStamp` (from base).
- `UVoxelInstancedStampComponent` (`VoxelInstancedStampComponent.h`) —
  `AddStamp(FVoxelStampRef) → int32 index`, `GetStamp(i)`, `SetStamp(i, ref)`,
  `RemoveStamp(i)`, `ClearStamps()`, `NumStamps()`, `UpdateStamp(i)`,
  `UpdateStamps(int32[])`, `UpdateAllStamps()`.

`FVoxelStampRef` is the universal handle. Concrete stamp types
(all under `Voxel/Public/...`) — each has a generated `_K2` library that exposes
`Make`, `Break`, `CastTo<Type>Stamp`, `MakeCopy`, and field getters/setters:

- `VoxelHeightmapStamp_K2.h` → `UVoxelHeightmap` asset → height layer (`Make` takes
  Heightmap, DefaultSurfaceType, WeightmapSurfaceTypes[], Layer, BlendMode,
  AdditionalLayers[], Transform, Behavior, Priority, Smoothness, MetadataOverrides,
  StampSeed, LODRange, bDisableStampSelection, bApplyOnVoid, BoundsExtension)
- `VoxelHeightStamp_K2.h` (base height stamp)
- `Shape/VoxelShapeStamp_K2.h`, `Shape/VoxelSphereShape_K2.h`,
  `Shape/VoxelCubeShape_K2.h`, `Shape/VoxelPlaneShape_K2.h`
- `Spline/VoxelHeightSplineStamp_K2.h`, `Spline/VoxelVolumeSplineStamp_K2.h`
- `Graphs/VoxelHeightGraphStamp_K2.h`, `Graphs/VoxelVolumeGraphStamp_K2.h`
- `StaticMesh/VoxelMeshStamp_K2.h`
- `Sculpt/Volume/VoxelVolumeSculptStamp_K2.h`, `Sculpt/Height/VoxelHeightSculptStamp_K2.h`
- `VoxelVolumeStamp_K2.h`

Universal stamp K2 (`VoxelStamp_K2.h`): `Get/Set Transform | Behavior | Priority |
Smoothness | MetadataOverrides | LODRange | BoundsExtension | StampSeed`,
plus `AddFloatMetadataOverride` and `AddColorMetadataOverride`.

### E. PCG voxel nodes — the actual class names (use these with `pcg.add_node` / `pcg.import_graph`)

All under `VoxelPCG/Public/`. The string to pass as `nodeType` is the UCLASS name
without the `U` prefix (so `PCGVoxelSamplerSettings` is the registered settings
class; PCG's node type list shows it under its `DisplayName`).

| Class | DisplayName | Key properties |
|---|---|---|
| `UPCGVoxelSamplerSettings` (`PCGVoxelSampler.h`) | "Voxel Sampler" | `Layer (FVoxelStackLayer)`, `PointsPerSquaredMeter` (0.1), `CellSize`, `Looseness`, `Tolerance`, `bApplyDensityToPoints`, `NewMetadatasToQuery[]`, `LOD`, `bResolveSmartSurfaceTypes`, `bUnbounded`, seed |
| `UPCGVoxelSamplerV2Settings` (`PCGVoxelSamplerV2.h`) | "Voxel Sampler Experimental" | `Layer`, `DistanceBetweenPoints` (100), `Looseness`, `MetadatasToQuery[]`, `LOD`, `bResolveSmartSurfaceTypes`, `bUnbounded`. **Note: V2 dropped PointsPerSquaredMeter for DistanceBetweenPoints.** |
| `UPCGVoxelStampSpawnerSettings` (`PCGVoxelStampSpawner.h`) | "Voxel Stamp Spawner" | `NewTemplate (FVoxelStampRef)`, `TargetActor (TSoftObjectPtr<AActor>)`, `SpawnedStampPropertyOverrideDescriptions[]`, `SpawnedGraphParameterOverrideDescriptions[]`, `PostProcessFunctionNames[]` |
| `UPCGCallVoxelGraphSettings` (`PCGCallVoxelGraph.h`) | "Call Voxel Graph" | `Graph (TObjectPtr<UVoxelPCGGraph>)`, `ObjectAttributeToType[]`, `bSynchronousLoad`. Dynamic pins driven by the chosen graph's parameters. |
| `UPCGCreateVoxelSplineSettings` (`PCGCreateVoxelSpline.h`) | "Create Voxel Spline" | `bVolumeSpline`, `HeightGraph` / `VolumeGraph`, `HeightBlendMode` / `VolumeBlendMode`, `Smoothness`, `Priority`, `Layer`, `StampBehavior`, `AttachOptions` |
| `UPCGVoxelElevationIsolines` (`PCGVoxelElevationIsolines.h`) | "Voxel Elevation Isolines" | `Layer (FVoxelStackHeightLayer)`, `LOD`, `ElevationStart/End/Increment`, `Resolution`, `bAddTagOnOutputForSameElevation`, `bProjectSurfaceNormal`, `bOutputAsSpline`, `bLinearSpline` |
| `UPCGVoxelProjectionSettings` (`PCGVoxelProjection.h`) | "Voxel Projection" | `Layer`, `KillDistance`, `bUpdateRotation`, `bForceDirection`, `LOD`, `MaxSteps`, `Tolerance`, `Speed`, `GradientStep`, `bQuerySurfaceTypes`, `NewMetadatasToQuery[]` |
| `UPCGVoxelQuerySettings` (`PCGVoxelQuery.h`) | "Voxel Query" | `Layer`, `HeightOrDistanceAttribute` ("QueryValue"), `bQuerySurfaceTypes`, `SurfaceAttributeSuffix`, `NewMetadatasToQuery[]`, `MetadataAttributeSuffix`, `LOD` |
| `UPCGWaitForVoxelWorldSettings` (`PCGWaitForVoxelWorld.h`) | "Wait For Voxel World" | Control-flow gate, no settings |
| `UPCGSpawnActorWithVoxelGraphSettings` (`PCGSpawnActorWithVoxelGraph.h`) | "Spawn Actor with Voxel Graph" | `TemplateActorClass`, `TemplateActor`, `PostSpawnFunctionNames[]`, `bForceDisableActorParsing`, `bInheritActorTags`, `TagsToAddOnActors[]`, `SpawnedActorPropertyOverrideDescriptions[]`, `SpawnedGraphParameterOverrideDescriptions[]`, `RootActor`, `AttachOptions`, `bSpawnByAttribute`, `SpawnAttribute`, `GenerationTrigger` |
| `UPCGApplyOnVoxelGraphSettings` (`PCGApplyOnVoxelGraphSettings.h`) | "Apply on Voxel Graph" | `ObjectReferenceAttribute`, `PropertyOverrideDescriptions[]`, `bSilenceErrorOnEmptyObjectPath`, `bSynchronousLoad` |

Voxel graph–side node types that produce points inside a `UVoxelPCGGraph` (not
PCG-graph nodes — these live inside a *Voxel* graph that PCG calls into):
`VoxelNode_GenerateSurfacePoints2D`, `VoxelNode_GetInputPoints`,
`VoxelNode_MergePoints`, `VoxelNode_PruneByDistance`,
`VoxelNode_RaymarchLayerDistanceField`, `VoxelNode_ScatterPoints`,
`VoxelOutputNode_OutputPoints`, plus `VoxelPointAttributeNodes`, `VoxelPointNodes`.
These authoring nodes aren't reachable via ue-mcp's PCG handlers — they need a
Voxel Graph–specific path, which ue-mcp currently does not have.

### F. VoxelGraph asset (the source-of-truth procedural graph)
`VoxelGraph/Public/VoxelGraph.h`. The asset class is `UVoxelGraph`; concrete
subtypes `UVoxelHeightGraph` and `UVoxelVolumeGraph` (in
`Voxel/Public/Graphs/`) — these are what a layer points at.

The only UFUNCTION here is editor-only: `SetBaseGraph(UVoxelGraph*)`. The rest
of the API (`AddParameter`, `RemoveParameter`, `AddTerminalGraph`, `FindParameter`,
`GetCompiledGraph`, …) is plain C++ — not callable through MCP without a new
bridge handler.

### G. Voxel graph parameter overrides (per-instance graph param edits)
`Voxel/Public/Graphs/VoxelParameterBlueprintLibrary.h` —
`UVoxelParameterBlueprintLibrary` with `K2_Get*`, `K2_Set*`, `Has*` for:
HeightGraph / VolumeGraph / HeightSculptGraph / VolumeSculptGraph parameters.
These are `CustomThunk` UFUNCTIONs (wildcard `int32&` value pin
backed by a custom thunk in C++) — they cannot be called over MCP through the
generic reflection path. **Wrapping them needs a dedicated bridge handler.**

### H. Misc
- `UVoxelTextureBlueprintLibrary::K2_CreateVoxelTextureFromRenderTarget(UVoxelTexture*&, UTextureRenderTarget2D*)` —
  convert an RT into a UVoxelTexture for use inside a graph (chain with Export above).
- `UVoxelPCGFunctionLibrary::GetPCGBounds()` — exposed only inside Voxel Graphs.
- Asset classes that are `BlueprintType` and only need standard ue-mcp
  asset.* tooling to create / property-edit: `UVoxelLayerStack`, `UVoxelHeightLayer`,
  `UVoxelVolumeLayer`, `UVoxelHeightmap`, `UVoxelMegaMaterial`,
  `UVoxelMetadata` (+ float/color/etc subclasses), `UVoxelStaticMesh`,
  `UVoxelTexture`, `UVoxelTool`, `UVoxelCharacter`, sculpt + spline + scatter
  graph assets, `UVoxelSurfaceTypeInterface`/`Graph`.

## Prioritized MCP-tool TODO

Format: each block is one shippable MCP action. **Tool name** is the prefixed
form ue-mcp would expose (e.g. `voxel_query_layer`). **Path** is the verb
written against ue-mcp tasks. **Implementation** is what the TS task does.

All of these are tractable today — they either compose existing ue-mcp tasks
or call a stable UFUNCTION via a single new path. None require us to invent C++.

### P0 — usable, no bridge work needed

**1. `voxel_world.get_status`**
Read voxel world health: `IsRuntimeCreated`, `IsVoxelWorldReady`,
`IsProcessingNewState`, `GetProgress`, `GetNumPendingTasks`.
- Path: 4× `level.get_actor_details` with `propertyName` filters, OR one
  `editor.invoke_function` per accessor. The cheap route is
  `editor.get_runtime_values` (it batches reads) once we know the actor
  label. Mid-effort variant: ask the bridge to add a single
  `voxel_world.snapshot` C++ handler that returns all five at once.
- Value: first thing any agent should be able to call. "Is the voxel world ready
  before I scatter into it?" answers itself.

**2. `voxel_world.create_runtime` / `voxel_world.destroy_runtime`**
Wraps the two zero-arg UFUNCTIONs on `AVoxelWorld`.
- Path: `editor.invoke_function` with `actorLabel`, `functionName="CreateRuntime"`.
- Value: scripted scene reset/regenerate.

**3. `voxel_world.set_layer_stack`**
Swap the LayerStack property on a voxel world actor.
- Path: `level.set_actor_property actorLabel propertyName=LayerStack value=<asset path>`.
- Value: pipeline trigger — load a biome, swap stacks, regenerate.

**4. `voxel_stamp_actor.spawn`**
Place a `VoxelStampActor` at a transform, with an optional stamp asset reference.
- Path: `level.place_actor actorClass=/Script/Voxel.VoxelStampActor location rotation scale label`. Optional follow-up `editor.invoke_function functionName="SetStamp"` once we know how to ship an `FVoxelStampRef` through JSON (see note below).
- Value: drop-in marker for designers to author terrain via stamps.

**5. `voxel_stamp_actor.update`**
Call `UpdateStamp()` on a voxel stamp actor by label.
- Path: `editor.invoke_function actorLabel functionName="UpdateStamp"`.
- Value: refresh after a property edit.

**6. `voxel_instanced_stamps.update_all` / `voxel_instanced_stamps.clear`**
Re-render or wipe a `UVoxelInstancedStampComponent` on an actor.
- Path: `level.set_component_property` to clear `PrivateStamps` (clear),
  or `editor.invoke_function` targeting the component for `UpdateAllStamps`.
- Value: bulk-stamp pipeline finalize.

**7. `voxel_pcg.build_scatter_graph`** *(replaces broken `voxel.scatter_meshes`)*
Use `pcg.import_graph` (the ue-mcp action that takes a JSON spec of
`{nodes, connections}`) to author a "Voxel Sampler → Transform Points
→ Static Mesh Spawner" graph in one call, then `pcg.set_static_mesh_spawner_meshes`
to populate weighted mesh entries.
- Params: `assetPath`, `meshes[{mesh, weight}]`, `pointsPerSquaredMeter?` (V1) or
  `distanceBetweenPoints?` (V2), `seed?`, optional metadata overrides.
- Implementation: emit a single `pcg.import_graph` call with three nodes
  (`PCGVoxelSamplerSettings`, `TransformPointsSettings`, `PCGStaticMeshSpawnerSettings`)
  and two edges. Use the **V1** sampler — V2 is "experimental" per its UCLASS.
- Value: the original promise of this plugin, finally working.

**8. `voxel_pcg.build_stamp_spawner_graph`** *(replaces broken `voxel.spawn_stamps`)*
Single `pcg.import_graph` call wiring `PCGVoxelSamplerSettings` →
`PCGVoxelStampSpawnerSettings` with the stamp template assigned and
TargetActor wired to a runtime variable.
- Params: `assetPath`, `stampTemplate (FVoxelStampRef payload — see note)`,
  `pointsPerSquaredMeter?`, `targetActor?`, optional
  `spawnedStampPropertyOverrideDescriptions[]`.
- Note on `FVoxelStampRef`: this is a struct whose JSON shape we have to
  inspect via `reflection.get_struct` once at design time. Realistically
  the v1 shape will be `{ stampType: "Heightmap" | "Shape" | …,
  heightmap?: "/Game/...", layer?: "/Game/...", transform?: … }` and
  TS code expands it before handing it to `pcg.set_node_settings`.

**9. `voxel_pcg.add_wait_for_world`**
One-call insert: take an existing PCG graph and prepend a `Wait For Voxel World`
node before a chosen sink. Voxel pipelines that scatter at editor time without
this run before the voxel world is ready and produce empty output.
- Params: `assetPath`, `sinkNodeName`.
- Implementation: `pcg.read_graph`, append `UPCGWaitForVoxelWorldSettings`
  node, splice the edge.
- Value: fixes the #1 footgun any new Voxel-Plugin PCG user hits.

**10. `voxel_pcg.build_isolines_graph`**
Build a graph that runs `UPCGVoxelElevationIsolines` and outputs either
points or splines.
- Params: `assetPath`, `layer`, `elevationStart/End/Increment`, `resolution?`,
  `bOutputAsSpline?`, `bProjectSurfaceNormal?`.
- Value: contour-line / road-layout generator. High novelty.

**11. `voxel_pcg.build_query_graph`**
Build a graph that runs `UPCGVoxelQuerySettings` to enrich an input
point set with attributes (height, surface type, metadata) — fan-in for
selection / colouring downstream.
- Params: `assetPath`, `layer`, `metadatas[]`, attribute names.
- Value: makes downstream `PCGAttributeFilter` selectors actually have
  data to filter on.

**12. `voxel_pcg.build_projection_graph`**
Wires `UPCGVoxelProjectionSettings` so an existing point cloud snaps to
the voxel surface.
- Params: `assetPath`, `layer`, `killDistance?`, `bUpdateRotation?`,
  `maxSteps?`, `tolerance?`.

**13. `voxel_pcg.build_create_spline_graph`**
Wires `UPCGCreateVoxelSplineSettings` to spawn `AVoxelHeightSplineStamp` /
`AVoxelVolumeSplineStamp` actors from a PCG spline.
- Params: `assetPath`, `bVolumeSpline`, `heightGraph?`/`volumeGraph?`,
  `smoothness?`, `priority?`, `layer?`, `stampBehavior?`.

### P1 — needs a small ue-mcp bridge addition

**14. `voxel.query_layer`** *(real `bake_heightmap` foundation)*
Synchronously sample heights at world positions through
`UVoxelQueryBlueprintLibrary_BlueprintOnly::QueryVoxelLayer` /
`MultiQueryVoxelLayer`. These functions have output reference params
(`bool& bSuccess`, `FVoxelQueryResult& Result`) — they cannot be called
through `editor.invoke_function` cleanly because the existing handler
returns a single JSON value, not multi-output reference params.
- Required bridge work: a new `VoxelHandlers::QueryLayer` /
  `QueryLayerMulti` in `UE_MCP_Bridge` that calls these directly, packs
  result into JSON, returns `{ value, normal, surfaceType, metadata: {…} }`.
- Value: this *is* the heightmap-bake feature, done right. Bake = grid of
  N×N MultiQuery calls → r16 file → `landscape.import_heightmap`.

**15. `voxel.export_to_rendertarget`** *(better `bake_heightmap`)*
Wrap `ExportVoxelDataToRenderTarget` — single call rasterises a 2D world-space
box of heights/metadata into a render target. Faster than QueryMulti for
heightmap baking, and the RT2D can then be exported via existing
`asset.export` (texture path).
- Required bridge work: an MCP action that
  (a) accepts an RT asset path + Layer + Box2D + Query spec,
  (b) creates the RT if absent,
  (c) calls `K2_ExportVoxelDataToRenderTarget`,
  (d) returns success and the RT path.
- Value: the proper, supported path to a heightmap.

**16. `voxel_sculpt.volume.*` / `voxel_sculpt.height.*`**
One action per UFUNCTION in the two sculpt libraries:
`paint_surface`, `flatten`, `smooth`, `sculpt_sphere`, `sculpt_cube`,
`sculpt_surface`, `sculpt_angle`, `apply_graph`, `clear_data`,
`clear_cache`, `get_save`, `load_from_save` (Volume), plus the height
variants `paint_surface`, `flatten`, `smooth`, `sculpt_height`,
`apply_graph`, save/load (Height).
- Required bridge work: a `VoxelSculptHandlers` family. Every function
  takes an actor label + a small payload (center, radius, strength, mode).
  These return `FVoxelFuture` — the handler waits with
  `Voxel::ExecuteSynchronously` (the same trick the K2 wrappers use).
- Value: scripted terrain editing — agents can carve roads, flatten zones,
  paint surface types. Big surface area, all small handlers.

**17. `voxel.stamp_ref.make_heightmap`** *(and friends per stamp type)*
Build an `FVoxelStampRef` payload from JSON and assign it to a
component / node settings property in one call. The ref struct is
non-trivial JSON — wrap once in a handler that mirrors
`UVoxelHeightmapStamp_K2::Make`. Worth doing for the half-dozen common
stamp types (Heightmap, Shape, Mesh, Graph) only.
- Required bridge work: one handler per stamp type, or a generic
  `voxel.stamp_make` that switches on a `type` field.
- Value: makes (4) and (8) above actually usable end-to-end.

### P2 — useful, larger scope

**18. `voxel_graph.set_parameter`**
Wrap `UVoxelParameterBlueprintLibrary::K2_Set*` (CustomThunk wildcards).
Can't go through `editor.invoke_function` — needs a dedicated handler that
reflects the parameter type via `FVoxelParameter` and packs the value
into an `FInstancedPropertyBag`.
- Required bridge work: one handler that takes
  `{ owner: actorLabel|assetPath, paramName, value }`, looks up the
  parameter type, constructs the value, calls the set.
- Value: live parameter tweaks for designers, scripted A/B sweeps.

**19. `voxel_graph.list_parameters`**
List of `{name, type, default, description}` exposed by a Voxel graph asset
or by a component owning one. Reads `UVoxelGraph::ForeachParameter`.
- Required bridge work: handler that walks the parameter map.

**20. `voxel_layer_stack.compose`**
Single-call builder: take an array of height-layer asset paths + volume-layer
asset paths, create a new `UVoxelLayerStack` asset, populate the arrays,
return the asset path. Existing `asset.create_data_asset` + `asset.set_property`
can do it; this is convenience and discoverability.
- Path: pure TS task composing existing ue-mcp actions.

**21. `voxel_pcg.build_call_voxel_graph`**
`UPCGCallVoxelGraphSettings` with a chosen `UVoxelPCGGraph` and a
`ParameterPinOverrides` propertybag. Dynamic-pin node, so the wrapper has to
read the graph's parameters first to know what to expose. Cleanly built once
(19) exists.

**22. `voxel_pcg.build_spawn_actor_graph`**
Wrap `UPCGSpawnActorWithVoxelGraphSettings` for "spawn an actor with a
voxel graph parameterized by point attributes" workflows.

**23. `voxel_texture.from_rendertarget`**
Bridge wrapper for `K2_CreateVoxelTextureFromRenderTarget` — needed to
close the loop on (15) when the destination is a `UVoxelTexture` asset
rather than just an RT.

## Notes on shipping path

- **Stop hand-rolling node graphs.** Replace every `pcg.create_graph` /
  `pcg.add_node` / `pcg.connect_nodes` chain in `src/tasks/` with one
  `pcg.import_graph` call carrying a JSON spec. The `shared/voxelGraph.ts`
  helper becomes obsolete.
- **`actionPrefix: voxel` is fine** but the current manifest categories are
  wrong: scatter/stamps belong in `pcg`, sculpting belongs in a *new*
  category (`sculpt`?), and heightmap export belongs nowhere near
  `landscape.bake_heightmap` because that task is fictional. Re-draft
  `ue-mcp.plugin.yml` after we pick which P0+P1 tools ship in v1.
- **Knowledge files (`knowledge/pcg.md`, `knowledge/landscape.md`)** are
  also wrong — they describe the imaginary tools, not real ones. Rewrite
  them after the tasks are real.
- **The `Voxel` plugin name is correct in `uePluginDependency:`** — the
  installed `.uplugin` at `../Vale/Plugins/Voxel/Voxel.uplugin` confirms it.
- **Citations to bring to the Voxel author next time.** When you re-show
  the plugin, every tool's description should cite the class + file it
  wraps. If we can't point to a UFUNCTION or a `pcg.import_graph` JSON
  shape with a known class string, we don't ship it.
